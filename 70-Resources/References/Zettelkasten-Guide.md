# Zettelkasten Method — Quick Reference

> How this vault implements the Zettelkasten method.

---

## Core Principles

1. **Atomicity** — One idea per note. If a note has multiple ideas, split it.
2. **Autonomy** — Each note should make sense on its own. No context needed.
3. **Linking** — Every note links to at least 2 other notes. No orphans.
4. **Own words** — Never copy-paste. Always rewrite in your own language.
5. **Emergence** — Structure emerges from links, not from folders.

---

## Note Types

| Type | Purpose | Location | Lifetime |
|------|---------|----------|----------|
| **Fleeting** | Quick capture | `00-Inbox/` | Hours to days |
| **Literature** | Process external sources | `20-Literature/` | Permanent |
| **Atomic** | One evergreen idea | `10-Atomic-Notes/` | Permanent |
| **MOC** | Organize topics | `30-MOCs/` | Evolving |
| **Journal** | Daily reflections | `50-Journal/` | Dated |
| **Project** | Active work | `40-Projects/` | Until complete |
| **AI** | AI synthesis | `60-AI/` | Review then promote |

---

## The Pipeline

```
Capture (Inbox) → Process → Create Note → Link → Review → Compound
     ↓                                                        ↑
     └──────────────── Weekly Review ────────────────────────┘
```

### Daily Flow
1. Capture everything into [[00-Inbox]]
2. Process inbox (aim for zero)
3. Create notes from processed items
4. Link new notes to existing ones
5. Write daily journal entry

### Weekly Flow
1. Review all daily notes
2. Extract insights into atomic notes
3. Run AI synthesis on the week's notes
4. Update MOCs with new links
5. Plan next week's focus
6. Archive completed items

---

## Linking Strategy

- **Related:** Ideas that are similar or complementary
- **Supports:** Evidence or reasoning that strengthens another idea
- **Contradicts:** Ideas that challenge or oppose each other
- **Extends:** Ideas that build on or deepen another idea
- **Source:** Where an idea came from

---

## When to Create a MOC

- You have 5+ notes on a similar topic
- You need a "table of contents" for a domain
- You want to see connections across sub-topics
- You're starting a new project or research area

---

## When to Archive

- Project is complete
- Note is superseded by a better version
- Topic is no longer active
- Yearly cleanup of old journal entries
